## 2. Architecture Overview

Tauri 2 desktop app. Rust backend (edition 2021) with a single
FK-enforced, additive-schema-only SQLite database per workspace
(`rusqlite`, bundled). Next.js/React/TypeScript frontend with Monaco and
xterm, communicating over typed Tauri `invoke()` bindings. Core safety
property: every autonomous change flows through
`requirement → task → execute → evidence → promotion` with mandatory
human approval before execution — this shape has not changed since its
introduction and is enforced by tests at every layer that touches it.

## 3. Completed Systems

Requirement Intelligence · Hash-chained Ledger + Evidence · Task DAG
Planning · unified Promotion Governance · Worker Intelligence (matching +
derived reliability) · Autonomous Reliability Layer (classification,
bounded retry, confidence, completeness, structured reports) · full UX
surface for all of the above (Governance panel, Workers panel, task
report drill-in) · production hardening (transactions, migration safety,
explicit FK enforcement, panic-safe read paths). All validated by
real-executor / real-`cargo check` tests — not mocked — per the
Sprint 10 release methodology recorded in
`docs/NEURALFORGE-V1.1-RELEASE-REPORT.md`.

## 4. Incomplete Systems

- Single worker/agent type (Coder only) — capability routing exists
  without a second type to route to.
- No DAG graph visualization (list/binding-only).
- No automatic retry-on-failure (on-demand by design, not a gap to
  silently close).
- No true simultaneous multi-file diff review UX.
- Chat-time automatic context injection is partial, not proven as a
  first-class feature.
- `docs/governance/Sprint-Gate-Protocol-v1.0.md` is an explicit
  placeholder awaiting human-supplied content.

## 5. Technical Debt

- `regenerate_history` is O(entries × evidence) with duplicated evidence
  blocks in its output — acceptable at current scale, flagged for cleanup.
- Dead `MODEL_FAILED` constant (`src-tauri/src/core/events.rs`) — the
  build's only compiler warning.
- Hand-written TypeScript types mirroring Rust structs across `lib/*.ts`
  — drift risk absent generated bindings (e.g. `ts-rs`).
- `run_code` tasks carry NULL correlation IDs (gating deferred since an
  earlier sprint).
- New worker profiles default to an optimistic 1.0 reliability score
  with no smoothing prior.
